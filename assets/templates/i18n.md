## Contributing

### App UI Translation

1. Copy `assets/bundles/en.json` → `assets/bundles/{code}.json`
2. Edit the header fields: `"lang"`, `"langCode"`, `"langRegion"`, `"langRegionCode"`
3. Translate every value — keys must remain unchanged
4. Submit a Pull Request

### Document Translation

The README and other docs are generated from locale-specific data files:

1. Copy `assets/docs/en.json` → `assets/docs/{code}.json`
2. Translate the values — keys must remain unchanged
3. Create `docs/{code}/` directory for any language-specific markdown files
4. Submit a Pull Request

The CI will automatically generate `docs/{code}/README.md` from your JSON + the shared `assets/templates/README.md` template.

### Adding a New Language

To add a new language, create both files:
- `assets/bundles/{code}.json` — App UI strings
- `assets/docs/{code}.json` — README content

Then trigger the CI by including `i18n` or `readme` in your commit message.
